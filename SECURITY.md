# Security Policy

## Supported versions

Only the latest commit on `main` is supported. No versioned releases.

## Scope

This app scrapes public government data and serves a static read-only catalogue. It holds no user credentials, payment data, or personal information. There is no login and no server-side session state.

If you find a way to inject malicious content into the medicine catalogue or the Cloudflare Pages deployment that could affect visitors, that is in scope.

## Reporting a vulnerability

Use **GitHub Security Advisories** — click "Report a vulnerability" on the [Security tab](https://github.com/chirag127/janaushdhi/security/advisories/new). Do not open a public issue for security reports.

Alternatively, email **chirag127** directly (address on GitHub profile).

Expected response: acknowledgement within 72 hours, fix or mitigation within 14 days for confirmed issues.
